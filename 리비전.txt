### Windows 버전(.exe) 빌드를 위한 단계별 안내

지금부터 **WSL2 터미널이 아닌, Windows의 터미널(PowerShell 또는 CMD)을 열어서** 다음 단계를 따라주세요.

1.  **Windows PowerShell 또는 CMD 실행:**
    *   Windows 시작 메뉴에서 `PowerShell` 또는 `CMD`를 검색하여 실행합니다.

2.  **프로젝트 디렉토리로 이동:**
    *   열린 터미널 창에 다음 명령어를 입력하여 프로젝트 폴더로 이동합니다. (Windows 경로 형식 사용)
    ```powershell
    cd D:\workspace_Gemini\자소서첨삭APP
    ```

3.  **Node.js 설치 확인 및 설치 (Windows용):**
    *   혹시 Windows에 Node.js가 설치되어 있지 않다면, 먼저 설치해야 합니다.
    *   **공식 사이트:** https://nodejs.org/ko
    *   위 사이트에서 **LTS 버전**을 다운로드하여 설치해 주세요.

4.  **Node.js 종속성 설치:**
    *   PowerShell 또는 CMD에서 다음 명령어를 실행하여 Windows 환경에 맞는 종속성들을 설치합니다.
    ```powershell
    npm install
    ```

5.  **Windows용 앱 빌드:**
    *   설치가 완료되면, 다음 명령어를 실행하여 `.exe` 파일을 빌드합니다.
    ```powershell
    npm run build
    ```
    *   이 명령은 `package.json`의 `win` 설정에 따라 Windows용 설치 파일을 `dist` 폴더 안에 생성할 것입니다.

6.  **설치 및 실행:**
    *   빌드가 완료되면, 파일 탐색기에서 `D:\workspace_Gemini\자소서첨삭APP\dist` 폴더로 이동합니다.
    *   그 안에 생성된 `.exe` 설치 파일을 더블 클릭하여 앱을 설치하고, 설치된 앱을 실행합니다.

이 단계를 거치면, WSL2의 복잡한 설정 없이 바로 Windows에서 실행되는 네이티브 앱을 얻을 수 있습니다. 이제 "워크스페이스 지정" 버튼도 정상적으로 동작할 것입니다.
