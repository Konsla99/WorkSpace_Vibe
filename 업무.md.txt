지금까지 논의한 디자인과 요구사항(폴더 선택, `cd` 이동, `gemini` 실행 등)을 실제 작동하는 윈도우 앱으로 구현하기 위한 **개발 지침 프롬프트**입니다.

이 프롬프트를 복사하여 새로운 대화창에 입력하거나, 저에게 그대로 명령하시면 각 파일의 전체 코드를 작성해 드릴 수 있습니다.

## 🚀 윈도우 앱 제작을 위한 개발 지침 프롬프트

**[프롬프트 제목: Electron 기반 Gemini CLI 내장 GUI 앱 개발]**

**1. 프로젝트 개요**

* **목표:** 사용자가 워크스페이스(폴더)를 선택하면 해당 경로에서 `gemini` CLI를 자동으로 실행하고 제어할 수 있는 윈도우 데스크톱 앱 제작.
* **핵심 스택:** Electron (Framework), Node.js, xterm.js (Terminal UI), node-pty (Terminal Engine).

**2. 기술적 요구사항 (구현 필수 항목)**

* **파일 탐색기 연동:** 윈도우 표준 폴더 선택 다이얼로그를 호출하여 선택된 폴더의 **절대 경로**를 가져올 것.
* **터미널 엔진 구현:** `node-pty`를 사용하여 `powershell.exe`를 백그라운드 프로세스로 생성하고, 초기 실행 시 사용자가 선택한 경로로 `cd` 명령어를 자동 전송할 것.
* **CLI 자동 실행:** `cd` 이동 직후 터미널에 `gemini`를 입력하여 CLI 환경을 즉시 활성화할 것.
* **UI/UX:** 제공된 다크 모드 HTML 디자인을 적용하되, 워크스페이스 미지정 시 "워크스페이스를 지정해 주세요"라는 검은색 플레이스홀더 화면을 유지할 것.
* **기능 버튼 매핑:** 사이드바의 5개 버튼 클릭 시 터미널에 직접 명령어를 입력한 것과 동일하게 작동하도록 구현 (예: `gemini --analyze-jd` 등).

**3. 생성해야 할 파일 리스트 및 역할**

1. `package.json`: 필요한 의존성(`electron`, `node-pty`, `xterm`) 및 빌드 스크립트 정의.
2. `main.js`: Electron 메인 프로세스. 시스템 다이얼로그 호출 및 터미널 프로세스(`pty`) 관리.
3. `preload.js`: 메인 프로세스와 렌더러(UI) 사이의 안전한 통신(IPC) 브릿지 설정.
4. `index.html`: 앱의 전체 레이아웃 및 CSS 디자인.
5. `renderer.js`: `xterm.js`를 초기화하고 메인 프로세스로부터 오는 터미널 데이터를 화면에 출력 및 사용자 입력 전달.

**4. 상세 기능 시나리오**

1. 사용자가 앱 실행 -> 검은 화면 중앙에 "워크스페이스 지정" 안내 노출.
2. "워크스페이스 지정" 버튼 클릭 -> 윈도우 탐색기에서 특정 폴더 선택.
3. 경로 선택 완료 시 -> 중앙 안내 문구 제거, `xterm` 터미널 UI 표시.
4. 자동 명령어 수행 -> `PS > cd "선택경로"` -> `PS > gemini` 순차 실행.
5. 사이드바 버튼 클릭 -> 현재 터미널 세션에 해당 기능 명령어 전송 및 결과 출력.
